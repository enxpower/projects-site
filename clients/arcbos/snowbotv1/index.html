ARCBOS™ 除雪机器人Snowbot v1.0 结构设计

冻结级 结构设计基线 Baseline v1.0

一台重载级、全电动、自主运行的除雪机器人。
紧凑型工业尺度，整体占地尺寸接近大型商用履带式除雪机。
该设备是工业机器人，而不是车辆。

【核心身份约束】
这是一个完全无人化的工业机器人系统。
在任何设计层面，都不允许承载、容纳或服务人类。
外观与结构必须明确避免任何“可驾驶设备”或“有人操作机械”的暗示。

【绝对禁止的元素】
- 任何形式的驾驶舱、座舱、操作舱
- 挡风玻璃、玻璃、窗户、透明面板
- 顶棚、车顶结构、雨棚
- 雨刷、后视镜
- 方向盘、座椅、把手、踏板
- 类车辆比例或车身语言
- 任何用于传感器或灯具的立杆、立柱、桅杆
- 科幻化造型、消费级产品风格、玩具感设计

【整体造型语言】
- 装甲化工业机器人平台
- 开放式结构或装甲钢壳式结构
- 平直、折线化或多面焊接钢板
- 清晰可见的螺栓、紧固件、焊缝、加强筋
- 加强型边角与抗冲击结构
- 可拆卸、螺栓固定的牺牲式耐磨部件
- 不存在任何足以容纳人体的封闭空间
- 明确呈现为“机器人/机械系统”，而非车辆

【工业设计哲学】
- 功能优先，零装饰
- 机械真实感：结构与受力路径可被直观看出
- 设计目标是承受冰块、路沿、硬雪、杂物的反复冲击
- 所有部件看起来都可维护、可更换
- 商用级工业设备，而非消费级产品

【大致尺寸比例】
- 宽度约 1.0 米
- 长度约 1.5 米
- 高度约 1.2 米
- 紧凑、密实的工业比例
- 低重心设计
- 短前后悬，无高耸上部结构

【底盘与材料】
- 重型焊接钢结构底盘
- 厚实钢板，清晰可见材料厚度
- 外露工业级紧固件
- 围绕除雪系统的加强型副车架
- 工业黄色 + 深灰色配色
  灵感来源于卡特彼勒（Caterpillar）与约翰迪尔（John Deere）
- 哑光或半哑光工业涂层
- 禁止高光、装饰性表面处理

【前端除雪系统】
- 商用级重载双级除雪系统
- 前置螺旋绞龙直接喂入中央叶轮
- 绞龙与叶轮处于同一连续壳体内
- 除雪流向在视觉与物理上必须真实合理
- 叶轮将雪垂直向上抛出
- 抛雪筒直接安装在叶轮壳体正上方
- 抛雪筒仅左右旋转，底座固定
- 抛雪筒为厚钢板结构，不得使用塑料外观
- 禁止侧向抛雪或不合理的气流路径

【切削边与耐磨件】
- 清晰可见、可更换的金属切削刀刃
- 螺栓固定的牺牲式切削边
- 硬化钢耐磨滑靴
- 耐磨件与结构件在视觉上有明确区分
- 重载商用级钢结构外观

【履带行走系统】
- 厚实的工业级橡胶履带
- 紧凑型履带驱动系统
- 相对于高度具有较宽的履带站距
- 针对积雪、结冰、不平整路面优化
- 不得出现轮胎
- 工业级驱动轮与导向轮部分外露

【动力与系统属性】
- 全电动驱动系统
- 电池供电的工业设备
- 不得出现排气口或暗示内燃机的进气结构
- 密封式电气舱
- 可隐约看到高压电安全警示标识

【传感器与自主系统】
- 激光雷达集成于低矮装甲化传感器舱内
- 工业相机嵌入装甲式传感器模块
- 传感器外置但与结构高度一体化
- 禁止玻璃罩、透明穹顶
- 禁止任何传感器立杆或高耸支架
- 传感器布局支持 360 度环境感知
- 所有感知硬件为工业级、抗冲击、抗低温设计

【安全与照明系统】
- 低矮圆柱形工业琥珀色 LED 警示灯
- 警示灯完全嵌入顶部装甲结构
- 禁止立杆、支架、外伸结构
- 短而宽、重载型警示灯
  风格接近机场或重型施工设备
- 前向高亮白色工业工作灯
- 后向琥珀色警示灯
- 侧向嵌入式示廓灯
- 所有灯具均为密封、防冲击、抗振动结构
- 适用于无人化、公共或半公共环境运行
- 符合商用设备可视与安全预期

【运行特征】
- 自主优先设计（Autonomous-first）
- 无任何明显人工操作界面
- 设计用于夜间、无人值守连续运行
- 工业可靠性高于外观美感
- 安静、电动、目的性极强的存在感

【场景与环境】
- 机器人正在商业停车场内执行除雪作业
- 暴雪环境，雪量大、雪花密集
- 雪流清晰可见地通过绞龙、叶轮与抛雪筒
- 夜间或低照度环境
- 工作灯与警示灯照亮飞雪颗粒
- 警示灯处于闪烁状态
- 场景中无任何人类
- 无近距离交互的车辆
- 电影级但真实的工业摄影风格
- 高真实度、物理逻辑合理
- 专业级商用工业设备气质

ARCBOS 硬件分阶段投入策略（BOM 负责人视角）

总体判断原则（先给结论）

Phase 1：凡是“换了就要重构系统”的 → 直接用顶级
Phase 2：凡是“接口稳定、性能可线性提升”的 → 允许升级
Phase 3：凡是“不影响安全与平台边界”的 → 延后

🟥 Phase 1【必须直接用顶级｜不能省钱】

这些硬件一旦选低配，后面升级 = 架构返工

1️⃣ 感知硬件系统（H5）—— ❗绝对第一优先级

H5. 感知硬件系统
├─ LiDAR（主环境感知）
├─ 工业相机（低照度 / 宽动态）
└─ 核心状态传感（IMU / 电流 / 温度）

原因（BOM 负责人判断）：

  • 这是“无人设备是否可信”的根基
  • 感知误差 → 不是性能问题，是安全责任问题
  • 后期更换 LiDAR = 重做标定、重做算法假设、重做验证

Phase 1 建议直接锁顶级：

  • Ouster / Hesai（工业级 LiDAR）
  • Basler / FLIR（工业相机 / 热成像）

✅ 结论：

Phase 1 必须买最贵，绝不妥协。

2️⃣ 计算与控制电子（H4）—— 平台神经系统

H4. 计算与控制电子
├─ 主控计算单元（Robot Computer）
├─ 实时控制单元（MCU / PLC）
└─ 工业 I/O 与现场总线

原因：

  • 决定你整个软件栈的“地基”
  • 算力/接口一旦变 → 软件架构跟着变
  • 这是平台，不是“可替换零件”

Phase 1 建议：

  • 工业级计算平台（即使算力富余）
  • 工业实时控制（不是消费 MCU）

顶流参考：

  • NVIDIA（Jetson 工业路线）
  • Beckhoff
  • Siemens

✅ 结论：

Phase 1 锁平台级配置，后面只能“加”，不能“换”。

3️⃣ 安全与可见性硬件（H7）—— 合规与责任边界

H7. 安全与可见性
├─ 工业警示灯 / 信号灯
├─ 工作照明
└─ 物理急停与安全回路

原因：

  • 无人设备能不能“被允许存在”
  • 这是法规 / 物业 / 客户第一眼看的东西
  • 后期升级意义不大，但返工成本高

顶流参考：

  • Federal Signal
  • Patlite

✅ 结论：

Phase 1 直接工业级顶配，一步到位。

4️⃣ 电池系统的“安全与低温部分”（H2 的关键子集）

H2（子集）
├─ BMS
├─ 低温加热与保护
└─ 高压保护与断路

原因：

  • 冬季 + 无人 + 大功率 = 风险叠加
  • BMS / 保护逻辑不稳 = 项目直接死亡
  • 电芯可以升级，但安全架构不能动

顶流参考：

  • CATL
  • BYD
  • 工业级 BMS（车规 / 工业）

✅ 结论：

Phase 1 用成熟方案，不做试验田。

🟨 Phase 2【允许升级｜性能驱动】

这些系统：Phase 1 用“够用 + 工业级”， Phase 2 再拉满性能。
