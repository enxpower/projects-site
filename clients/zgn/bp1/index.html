闲时算力调度平台商业计划书

第一章 执行摘要（修订版）

随着人工智能浪潮席卷全球，算力已成为国家战略资源和数字经济的核心生产力。中国智算中心建设进入高速发展期，但仍存在算力资源异构纳管难、算网割裂协同难、算力供需错配调度难等痛点 闲时算力调度平台商业计划书-v01。

“神算子”闲时算力调度平台应运而生。我们是一家专注于智算产业生态的创新企业，通过“硬件销售 + 算力运营”的双轮驱动模式，力争成为领先的智能算力资源整合与调度平台。

我们的核心亮点

  1. 硬件渠道资源优势：与 NVIDIA、华为、DDN 等厂商官方渠道绑定，确保设备交付稳定与价格优势。
  2. 绿色低价能源结合：重点布局新疆、内蒙古等低成本绿色电力区域，电力成本较东部平均低 30%–40%。
  3. 先进的调度与池化技术：支持跨域异构闲置算力整合、显存回收与动态调度，显著提升资源利用率。
  4. 价格与SLA优势：在同等 SLA 情况下，价格可较主流公有云低 30%–40%，兼顾灵活性与稳定性。

市场规模与增长路径

  • TAM：中国 AI 推理与中小科研弹性算力市场（年支出规模，百亿级）。
  • SAM：聚焦新疆、内蒙、京津冀、华南等区域可接入的智算中心资源。
  • SOM：36 个月内可服务市场（按 GPU 数 × 利用率 × 单价测算）。

预计未来 3–5 年内，闲时算力市场将保持 40%+ 年复合增长率，平台业务收入将在第 3 年超过硬件销售，成为主要增长引擎。

第二章 市场与政策分析（修订版）

2.1 宏观政策背景

中国正加快建设全国一体化算力网络，政策层面已形成“三横四纵”的国家枢纽体系。东数西算、智算中心建设、AI 推理场景开放等一系列举措，为闲时算力平台创造了供给端充裕 + 需求端爆发的政策窗口。

政策要点

  • 《新型数据中心发展三年行动计划》：推动算力利用率提升，要求 PUE ≤ 1.3。
  • 《东数西算工程》：推动西部绿色电力算力向东部需求地输送。
  • 地方示范项目：上海、深圳、合肥等城市已上线政务/科研 AI 平台试点。

2.2 市场规模（TAM/SAM/SOM）

  • TAM（总可用市场）：中国 AI 推理 + 中小科研弹性算力年支出约 300–400 亿元。
  • SAM（可服务市场）：结合区域（新疆、内蒙、京津冀、华南）+ 模型生态 + 网络互联限制，约 80–100 亿元。
  • SOM（可获取市场）：按“36 个月接入 GPU 数 × 平均利用率 × 单价 ¥/GPU-hr”测算，约 20–30 亿元。

2.3 行业趋势

  1. 需求侧：
    • 推理需求快速超过训练（占比由 40% → 65%），适合夜间/可抢占实例。
    • 政企客户开始批量采购 AI 推理服务，预算科目归类为“数据中心服务/信息化专项”。
  2. 供给侧：
    • 全国已建/在建智算中心 100+，GPU 保有量超百万片，闲时利用率不足 50%。
    • 西部地区电价低至 0.2–0.3 元/kWh，具备算电协同套利空间。
  3. 价格带：
    • 公有云竞价实例价格 ¥25–40/GPU-hr；
    • 闲时算力平台可稳定比肩 SLA 下低 30–40%。

2.4 客户场景与采购路径

第三章 痛点与机会（修订版）

3.1 供需错配

当前中国智算中心 GPU 数量快速扩张，但供需存在严重错配：

  • 峰谷不均：日间/训练任务集中，GPU 高负载；夜间/周末则闲置率超过 40%。
  • 地域差异：西部电价低但需求弱，东部需求旺但电价高、资源紧张。
  • 合规约束：部分跨域/跨境算力无法调度，导致“有闲不用”。

中文读图说明

  • 横轴：一天的 24 小时。
  • 纵轴：GPU 利用率（%）。
  • 早晨 6 点前：处于低谷（约 20–30%），资源大量闲置。
  • 上午 10–12 点：利用率达到峰值（约 75–80%），主要因训练与批量任务集中。
  • 下午与傍晚：维持在 60–70% 的高位。
  • 夜间 22 点后：需求回落（约 50–55%），出现明显闲时算力。

3.2 成本刚性

  • 公有云价格带：¥25–40/GPU-hr，波动小，长期维持高位。
  • 底层成本拆解：电力（约 ¥6–8/GPU-hr）、折旧（¥8–10/GPU-hr）、网络与出网（¥3–5/GPU-hr）、运维与分成（¥2–4/GPU-hr）。
  • 问题在于：用户承担的单价远高于底层边际成本，缺乏灵活性。

  • 绿色柱 (Our Price)：平台对外售价，约 ¥30/GPU-hr。
  • 蓝/红/紫/橙色柱 (Alibaba / Huawei / Tencent / AWS China)：主流公有云厂商的 GPU 定价，多在 ¥33–38/GPU-hr 区间。

3.3 用户体验痛点

我们调研发现，科研与政企客户最关注的并非单价，而是算力可用性与交付体验：

  • 排队时延：在主流公有云，低优先级任务平均排队 2–6 小时；科研项目常因延迟影响实验进度。
  • 任务失败率：因节点抢占/迁移，失败率可达 1%–2%，用户需要重复提交。
  • 环境部署时延：镜像启动平均 15–30 分钟，对短时推理任务极不友好。

  • 峰谷错配转化率：夜间可迁移任务占总推理需求 25–30%，具备规模化调度价值。
  • 政企与科研客户：预算刚性、合规优先，愿意为稳定 SLA 付费。

第四章 解决方案（修订版）

4.1 平台定位

“神算子”平台定位为全国闲时 GPU 调度与资源整合平台，通过跨地域池化、动态调度、任务迁移，将分散的闲置算力统一成对外可用的服务，实现 “以算代储 / 算电协同” 的新型能源与算力结合模式。

4.3 算电协同机制

  • 低谷套利：在西部绿电区域，低谷电价 0.2–0.3 元/kWh，结合 PUE=1.3，可实现 单位算力成本降低 30%+。
  • 跨域迁移：通过闲时调度，把推理类任务从东部高电价时段迁移至西部低电价时段。
  • 收益模式：节省的电力成本部分让利客户，部分形成平台毛利。

读图说明

第五章 技术与架构（修订版）

5.1 技术总体思路

平台以“调度—计费—观测”为核心闭环，实现跨地域异构 GPU 的池化调度。通过自研调度引擎 + 合作方虚拟化/容器化组件，实现算力资源的统一管理与智能分配。

5.2 平台架构设计

平台架构分为三层：

  1. Northbound（北向接口）
    • API/SDK：标准化 API，兼容主流 AI 框架。
    • 作业编排：支持 K8s/Slurm/Ray 等调度方式。
    • 镜像仓库：容器镜像与模型权重托管。
  2. Control Plane（控制平面）
    • 资源发现：跨域 GPU 统一注册。
    • 调度器：基于约束/打分/反亲和的智能调度。
    • 计费与风控：GPU-hr 精准计量、信用闸门、异常检测与赔付。
    • 安全审计：密钥管理、操作留痕、跨域权限。
  3. Data Plane（数据平面）
    • 多地域集群：支持东西部跨域。
    • 网络与加密：TLS/零信任接入。
    • 数据脱敏：敏感数据隔离与合规处理。
    • 故障注入与熔断：确保跨域容灾。

12.1 核心风险识别

📊 [TAB-10 风险红线指标表]

12.2 风险监控与触发机制

我们将这些风险纳入季度董事会报告，并建立实时监控：

  • 供货风险：监控 GPU 采购合同与交付周期 → 若 >60 天，立即调整采购渠道并启用备份机型。
  • 现金流风险：设定月度回款日报 → 若回款周期超 60 天，立即暂停赊销并启动保理/保函。
  • SLA 风险：设定 SLA 实时面板（排队时延、失败率、迁移时延） → 若违约率 >0.5%，触发容量扩容与调度优化。
  • 合规风险：定期审计日志 → 若发现违规操作，立即暂停相关客户接入，启动合规整改。

📊 [FIG-15 风险监控流程图]

12.3 兜底预案与缓释手段

  • 供货：签署多渠道供货协议（国内外厂商并行），建立“以销定采 + 期货锁单”机制。
  • 现金流：引入包年包月预付费模式，降低应收账款占用；在必要时，利用融资额度建立流动性缓冲。
  • SLA：保留 10%–15% 的闲置冗余容量，用于故障切换；季度级混沌演练，保证灾备能力。
  • 合规：成立合规专责小组，确保通过等保三级/密评；在敏感区域任务启用数据脱敏与跨境隔离策略。

12.4 投资人保护机制

我们将关键风险纳入投后管理，建议在融资协议中设置可量化的对赌/转换条件：

  • 平台 SLA 达成率 ≥99.9%
  • 平均回款周期 ≤45 天
  • 平台收入占比 ≥30%
