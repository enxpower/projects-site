🟥 **A.1 地面机器人 MVP（除雪/除草）

—— 商业真正能跑起来的“最小闭环版本”**

目标：第一年就能卖出去、能真实替代人工、能赚钱，而不是“演示用玩具机器人”。

这是你整个机器人事业的第一支“现金流武器”。

我会从工程、AI、商业、运营四个维度拆解 MVP。

🟦 一、MVP 必须解决的核心痛点（按优先级）

1）能确切把任务区清干净（基本清扫能力）

  • 积雪厚度：0–30 cm（MVP 不要挑战 50 cm 大雪，那是 Phase 2）
  • 地面类型：水泥、沥青、停车场标线
  • 清扫宽度：50–70 cm（最具性价比）
  • 机械结构要稳、推力足、不会被卡住

如果清扫不干净，所有 AI 全废。

2）能安全地自动绕开障碍物（安全优先）

  • 自动绕车
  • 自动绕人
  • 自动避开路沿
  • 不撞岗亭、灯杆

这是客户最看重的：

“别撞车、别伤人、别出事故。”

3）能从 A 点 → B 点执行“完整任务闭环”

也就是：

  • 自主发车
  • 清扫路径
  • 中途避障
  • 自动补电
  • 回到起点
  • 任务结束自动报告

这叫 闭环任务。

只要做到这一个闭环，客户就能买。

4）能让普通物业人员 5 分钟上手

机器人不是给工程师用，是给物业大叔用。

必须做到：

  • 一键启动
  • 一键回桩
  • 人可以远程接管
  • 手机 App 就能看状态
  • 所有复杂操作自动化在后台

好用，就是竞争力。

5）成本必须中国化（供应链优势）

这一点你最擅长，中国制造是你唯一能碾压北美同行的武器。

MVP 选型：

  • 电机：国产轮毂电机
  • 电池：国产刀片/三元
  • 驱动器：国产无刷驱动
  • 结构件：钣金 + 铝型材
  • 激光雷达：禾赛 / 镭神（低配即可）
  • 摄像头：海康组件

整机 BOM 控制在 6,500–8,000 美元

整机售价：18,000–30,000 美元

→ 足够盈利，而且比欧美同行便宜 50%。

🟩 二、除雪/除草 MVP 的“极简技术方案”（最稳妥的路线）

我给你一个非常现实、最低风险、最快落地的方案。

1）导航系统：激光雷达 + UWB（非 RTK）

为什么不是 RTK？

因为 RTK 在停车场和建筑旁边极不稳定。

你的 MVP 不需要厘米级精度，

只需要：

  • 激光雷达 SLAM（导航）
  • UWB（提供信标，增强定位）

这是最稳最不掉链子的组合。

2）避障系统：三层冗余

  • 主：激光雷达
  • 辅：前向摄像头（识别人/车辆）
  • 保底：触碰条（物理急停）

这是必需的安全底线。

3）任务规划：人工画路线 + 自动补全

MVP 不要搞“自动生成路线”，太难、太慢。

正确方式：

1）物业人员用 App 手画路线

2）机器人自动补齐可扫区域

3）执行时自动避障、自动重规划

就这么简单，客户就够用了。

4）补电方式：回桩充电（自动对准）

  • 不需要自动换电
  • 不需要机械臂
  • 只需自动对准触点就够了

这是最成熟、风险最低的方案。

5）状态管理：云端 + 本地边缘一起

  • 云端：任务调度、记录、日志
  • 本地：断网时仍能跑闭环

北美物业的网络烂得惊人，

本地闭环必须可靠。

🟥 **A.2 安防机器人 MVP（Security Patrol Robot）

——“能真正替代夜班保安”的版本**

MVP 目标（一句话）

一个能够自主巡逻、自动上报事件、自动充电、能远程接管的夜间安保机器人。

🟦 一、安防 MVP 必须满足的核心功能（按优先级排序）

1）可靠的巡逻路线执行（核心任务）

  • 按固定时间巡逻固定路线
  • 自动绕障
  • 自动补充遗失路线
  • 自动返回充电桩

只要能一直巡逻，就是成功的 50%。

2）事件检测能力（这是商业价值核心）

MVP 的目标不是“检测所有事件”，

而是识别 5 种 最关键、最高发生率、最高决策价值 的事件：

  1. 可疑人物（凌晨 1–5 点）
  2. 禁止区域闯入
  3. 车辆异常停放
  4. 摔倒 / 异常动作（员工园区）
  5. 烟火源 / 火焰（高风险区）

这 5 类事件，占全部物业安防事件的 80%。

MVP 不要追求“全事件检测”，那是 Phase 2。

3）实时视频回传 + 双向语音

你必须做到：

  • 保安中心能看到实时画面
  • 能远程喊话
  • 能远程接管路径

否则无法真正替代“人类保安的功能”。

4）全天候运行（夜间为主）

重点是：

  • 夜间红外/低照度摄像头
  • 自动充电
  • 8–12 小时续航
  • 雨雪天气可运行（防水 IP 控制）

夜巡是机器人的绝对强项，因为：

  • 人类保安最容易犯困
  • 人类保安最容易漏巡
  • 人类保安最害怕夜间独自巡逻

机器人完全没有这些问题。

5）安全冗余与监控能力

客户关心“事故责任”。

所以 MVP 必须：

  • 配置激光雷达（主避障）
  • 配置摄像头辅助避障（识别黑衣人）
  • 配置保险杠触碰条
  • 配置蜂鸣警告灯

MVP 不需要做全自动室内外切换。 MVP 只做户外固定园区巡逻。

🟩 二、安防机器人 MVP 的“极简技术方案”

1）定位：激光雷达 SLAM + 惯性 + UWB（选配）

不做 RTK（不稳定）。

不需要高精度，只需要可靠性。

2）识别能力：轻量级 AI 模型（边缘运行）

你不需要大模型。

MVP 用：

  • 人形检测
  • 动作异常检测
  • 火焰检测
  • 车辆识别
  • 越界检测（基于虚拟围栏）

全部在边缘设备运行，延迟低、稳定。

3）通讯链路：4G/5G + 局域 WiFi（回桩时上传大数据）

夜巡视频回传并不需要很高带宽。

机器人必须支持：

  • 在线实时视频
  • 弱网本地缓存
  • 回桩 WiFi 批量上传

这是成熟方案。

4）充电方式：自动定位对准桩（与除雪同一平台）

你可以共用同一自动充电桩系统：

  • 对准精度 ±5 cm
  • 双弹片触点
  • 上方定位摄像头辅助

成本低、稳定、可量产。

5）底盘设计：慢速、安全为主

安防机器人不需要跑快。

  • 巡逻速度：0.5–1.2 m/s
  • 爬坡能力：10°
  • 防水：IPX5
  • 工作温度：-20~45°C（北美刚刚好）

结构尽量简单，坚固，适合户外。

🟥 A.3 工业巡检机器人 MVP（地面巡检）

——替代“人工每天走一遍”的闭环巡检机器人

📌 MVP 目标（一句话）

能够自主巡检固定路线、自动生成巡检报告、检测关键资产异常，并在断网情况下仍能安全闭环运行。

巡检不是炫技，而是一个 流程管理问题。

你要做的是一台能够每一天毫不喊累、毫不漏项地完成巡检任务的机器。

🟦 一、地面巡检的 MVP 必须做到哪些（按优先级排序）

1）路线巡检（核心中的核心）

机器人每天照着固定路线走，完成固定任务。

这必须做到：

  • 路线精准执行
  • 不掉线、不乱跑、不迷路
  • 自动记录每一个点的巡检数据
  • 任务结束自动回桩充电

路线巡检 = 价值 80%

这就是为什么 Boston Dynamics 的 Spot 能赚钱——不是因为跳舞，是因为巡检路线稳定。

2）资产点检测（最基本的巡检能力）

MVP 不要追求“全功能巡检”。

你只需要覆盖工业客户最关心的 五类基础异常：

Top 5 工业巡检异常（覆盖率 >80%）

  1. 温度异常（热像仪）
  2. 冒烟 / 火焰
  3. 液位异常（视觉）
  4. 压力表读数（视觉 OCR）
  5. 门/柜/阀处于错误状态（闭/启）

这五类足够让石油、光伏、变电站、工厂买单。

一定不要一开始做复杂操控，比如开关阀门或现场修理 —— 那是 Phase 3。

3）事件上报（工业巡检最重要成果）

客户要的是：

  • 异常识别
  • 自动截图
  • 自动生成报告
  • 自动存档
  • 自动推送到邮箱/运维系统

巡检是“文字和证据工作”，

而不是“炫技动作”。

4）极高的可靠性（工业客户最看重）

MVP 不要求高智商，但必须：

  • 不漂移
  • 不翻车
  • 不陷坑
  • 不掉电
  • 不死机
  • 不乱撞

工业巡检最大的风险就是机器人宕机导致生产事故无法及时发现。

你的机器人必须有：

  • 机械稳定底盘（四驱/六驱）

🟥 **D.2 模块二：UAV → Robot 任务投放系统

（Aerial Task Dispatch，简称 ATD）**

这是你最核心的创新，别的机器人企业都没有。

📌 UAV 为什么能给机器人“布置任务”？

因为 UAV 具备：

  • 全局视角
  • 边界扫描能力
  • 大范围资产检测能力
  • 异常识别能力

地面机器人无法获取这些数据。

因此 UAV 可以：

  • 标记积雪 / 草长 / 积水区域
  • 标记可疑人 / 车辆
  • 标记巡检异常热点
  • 标记障碍位置

这些标记位置 → 送入 OS → OS 决定配送给哪个 robot。

📌 ATD 的执行流程具体如下：

1）UAV 完成一次扫描（Coverage Flight）

将结果上传（2D/3D + 分割掩模 + 热像图）

2）AI 模型对扫描结果进行分析

提取：

  • 异常点（asset abnormal）
  • 清扫需求区（snow/dirt/grass mask）
  • 可疑行为（security event）
  • 阻塞点（blocked path）

3）Mission Scheduler 生成任务

例如：

  • “区域 B6 积雪厚 15cm”
  • “区域 D4 发现三人停留超过 2 分钟”
  • “区域 A3 的变压器温度偏高”

4）任务按 Robot 类型分发

  • 除雪机器人获得“雪区任务”
  • 安防机器人获得“可疑活动任务”
  • 巡检机器人获得“资产异常任务”

5）Robot 接任务 → 执行 → 回传结果

  • 机器人执行
  • UAV 二次确认
  • 地图打印更新

6）最终闭环：

UAV 扫描 → OS 分配 → Robot 执行 → UAV 验证 → 地图更新

这是 物理 AI 的闭环智能链路。

🟥 **D.3 模块三：Robot → UAV 异常召唤系统

（Ground-to-Air Call，简称 GAC）**

Robot 可以召唤 UAV。

这是空地一体的第二大创新。

📌 Robot 为什么要召唤 UAV？

因为地面机器人有视野盲区、环境限制：

  • 停车场出口突然大量来车
  • 草坪迷路、路线受阻
  • 安防机器人发现可疑人物但无法跟踪
  • 巡检机器人发现气体泄漏，但看不清全局
  • 除雪机器人发现巨大障碍阻路

这些情况需要：

“无人机飞来补盲，提供另一维度的信息。”

📌 GAC 触发机制（非常重要）

触发类型：

  1. 异常事件触发

    安防机器人：可疑人

    巡检机器人：烟雾/热像异常

    除雪机器人：大障碍/危险

  2. 困境触发

    地面机器人卡住/路线断开/大范围阻塞

  3. 低信心触发（AI Confidence Drop）

    机器人本地“看不清”，需要 UAV 提供高空辅助判断

  4. 任务确认触发

    机器人完成任务 → UAV 来验证结果

    （非常适合巡检 & 安防 & 除雪）

📌 GAC 的执行流程：

  1. 地面机器人触发 GAC
  2. OS 下发“空中快速盘旋任务”给 UAV
  3. UAV 抵达空域
  4. UAV 上传鸟瞰视频/热像/深度
  5. OS 再次判断 → 提示 robot 执行下一步

这就是完整的“上下文补盲”链路。

🟥 N.1 技术风险（Technical Risks）

风险 1：空地一体化协同调度的系统复杂度高

风险描述：

空地协同涉及机器人定位、UAV 探测、任务图谱、异构算力调度、实时地图更新，系统工程复杂度高。

后果：

  • 开发周期延长 6–12 个月
  • 可靠性不足时会影响客户试点

解决方案：

  • 用 最小可行闭环（MCL） 方式逐步升级
  • 第一阶段：地面机器人独立执行 100%，无人机可选加速
  • 第二阶段：无人机扫描 → 地面机器人成队协同
  • 第三阶段：空地协同 + 事件识别 + Mission Graph 全链路闭环

可量化指标：

  • Year 1：地面独立可靠性 + UAV 地图输入
  • Year 2：空地联合任务成功率 ≥ 97%
  • Year 3：自治级别达 Level 4（多机器人 + 多 UAV 同时调度）

风险 2：机器人 SLAM + 冬季恶劣环境表现不稳定

风险描述：

雪地、冰面、反光、极端低温会造成 LIDAR、RTK、IMU 的退化。

后果：

  • 降低任务完成率
  • 无法稳定商用部署

解决方案：

  • 用 “多模态传感器冗余”：LIDAR + 超声 + 毫米波 + 视觉
  • 增加 雪地特化算法模型库
  • 采用 轮毂电机 + 防滑轮 + 链条结构

可量化指标：

  • 20℃ 下定位精度误差 < 3cm
  • 雪地任务成功率 ≥ 98%

风险 3：无人机与地面机器人的通信链路不稳定

风险描述：

长距离、遮挡、金属厂区可能导致链路不可靠。

后果：

  • 协同任务失败
  • 无法满足企业 SLA

解决方案：

  • 三链路冗余：5G / WiFi6 / 900MHz
  • Mission Graph 使用 断链继续执行 的自治策略
  • UAV → OS → Robot 采用发布–订阅模式，不依赖持续连接

可量化指标：

  • 数据链路 uptime ≥ 99.5%
  • 异构机器人“断链继续”成功率 ≥ 90%
