EnergizeOS™ 微电网EMS系统功能规划与产品开发计划

一、总体产品架构原则

本产品自设计之初即确立为全球统一内核、区域能力插件化的微电网 EMS 架构。

所有市场共用单一 Global Core，确保控制逻辑、系统行为与工程边界的一致性；各区域差异仅通过参数、接口与策略模块实现，不允许演化为分叉代码库。

核心原则

  • 单一全球核心代码库（Global Core）
  • 区域能力以模块 / 插件 / 配置方式加载
  • 核心控制逻辑不因区域改变
  • 区域模块独立为可选 SKU / 订阅项，不侵入核心

该架构直接服务于三项核心目标

  • 长期工程可维护性：避免多区域并行演进导致的失控复杂度
  • 产品可复制性：同一核心快速适配不同市场与项目规模
  • 商业可定价性：形成清晰的 Base + Add-on 产品结构

二、全球核心功能（Global Core）

所有区域 必须具备，构成 EMS 的最低可行产品（MVP）

1️⃣ 并网 / 离网 / 再并网控制（Grid ↔ Island）

功能范围

  • 主网异常检测 → 自动孤岛
  • 主断路器控制（跳 / 合）
  • 同步检查与重联控制
  • 黑启动流程（无外部电源）

工程要点

  • 明确合闸唯一授权路径
  • 支持多步骤状态机（而非简单逻辑）

2️⃣ 多电源经济调度（Economic Dispatch）

调度对象

  • 光伏 / 风电
  • 储能（BESS）
  • 电网进出功率
  • 柴油 / 燃气发电机

核心算法约束

  • 电价 / 燃料成本
  • 电池 SOC / 寿命约束
  • 备用容量
  • 排放或运行限制

3️⃣ 关键负荷管理（Critical Load Management）

功能范围

  • 关键负荷清单与优先级
  • 孤岛 / 电源不足时的自动减载与恢复
  • UPS 协同，保证关键回路 0 中断

工程要点

  • 负荷必须可分级、可回溯
  • 减载与恢复需有完整日志

4️⃣ 频率 / 电压协调控制（Stability Coordination）

并网模式

  • 按调度指令控制功率交换
  • 遵循外部网规

孤岛模式

  • 下达 P/Q 设定值
  • 协调 droop / 下垫控制
  • 稳频稳压

注意

  • EMS 是“协调者”，不是一次控制器
  • 必须适配不同 PCS / 发电机控制器能力

5️⃣ 储能管理（BESS Management）

功能范围

  • SOC / SOH 监控
  • 备用 SOC 保留策略
  • 电池均衡 / 循环策略

技术适配

  • 锂电
  • 液流电池
  • EV / V2G

6️⃣ 监控、告警与审计（Monitoring & Audit）

必需能力

  • 实时状态监控
  • 多级告警
  • 事件 / 操作日志（不可篡改）
  • 历史数据与趋势
  • 报表导出

安全

  • 用户权限分级
  • 操作留痕
  • 基础网络安全防护

7️⃣ 通信与设备接入（Connectivity）

南向

  • Modbus TCP / RTU
  • IEC 61850
  • DNP3
  • IEC 104 / 101

北向

  • SCADA / DMS
  • 云平台（可选）

工程要求

  • 标准化设备驱动层
  • 不与具体厂商深度绑定

8️⃣ 用户界面与运维（HMI & O&M）

形态

  • 本地 HMI
  • Web
  • 移动端（可选）

功能

  • 拓扑可视化
  • 参数配置
  • 远程运维（权限隔离）

五、总结

本 EMS 产品以“全球统一控制内核”为基础，覆盖并离网切换、多电源调度、关键负荷与稳定控制等通用能力；各区域的监管、市场和应用差异通过可选插件实现，从而在保持工程一致性的同时，实现快速区域适配与模块化商业变现。

六、EMS 产品里程碑规划（V1 → 可商用）

V1 总体开发路线（0–12 个月）

一句话定位：

V1 不是“全能 EMS”，而是一款 可部署、可审计、敢进现场的商用微电网控制内核。

V1 范围先行冻结（避免发散）

目标市场优先级

  • 主攻：商用 / 工商业园区微电网（500kW–50MW）
  • 次选试点：
    • 美国：商业 / 校园
    • 澳大利亚：矿区 / 牧场（仅选可复用能力）
  • V1 明确不做（统一放入 V2+ 插件）：
    • 欧洲能源社区结算
    • 东南亚预付费体系
    • 复杂辅助服务市场结算

V1 必须交付的能力集合

Global Core（必须全量）

  • 并网 / 离网切换控制
  • 经济调度（规则优先）
  • 关键负荷保障
  • 稳态控制（频率 / 电压）
  • BESS 管理
  • 监控、告警、事件、审计
  • 通信抽象
  • 本地 HMI / 运维界面

最小插件框架（为未来留接口）

  • 网规与参数配置化
  • 设备驱动模块化
  • 标准接口预留（OpenADR / IEC 104 等）

区域示范（只选 1 个）

  • 方案 A：美国 IEEE 1547 参数模式
  • 方案 B：中国 TOU / 峰谷电价表

二选一，随首个试点决定，不做并行。

阶段一：核心原型（0–3 个月）

关键词：能跑、能切、能追溯

运行平台与硬件基线

  • 工控机 / 工业控制器 + I/O 定版
  • 本地优先、离线可运行的 OS / 控制框架
  • 通信与安全最小基线（账户雏形、最小端口暴露）

数据与设备抽象

  • 统一数据模型（测点 / 状态 / 告警 / 事件 / 命令）
  • Modbus 仿真：PV / BESS / DG / PCC / 负荷
  • 采集 → 判据 → 控制 → 反馈 的完整闭环

并离网状态机（V1 核心）

  • 主网异常检测 → PCC 跳闸
  • 孤岛模式切入（BESS / DG）
  • 再并网同步判定（频差 / 相位 / 电压窗）→ 合闸

关键负荷最小机制

  • 关键负荷清单与优先级
  • 负荷切除 / 恢复命令链路（允许虚拟）

HMI 最小闭环

  • 实时状态：V / I / f / P / SOC、PCC、运行模式
  • 告警：失电、切换中、重联失败
  • 手动 / 自动切换 + 基础权限

测试环境

  • 数字仿真模型（PV + BESS + DG + 负荷 + PCC）
  • 核心场景脚本：
    • 失电
    • 孤岛稳态
    • 再并网失败与回退

阶段验收底线

  • 失电检测 → 跳闸 ≤ 1s
  • 并网判据可配置、可记录
  • 全流程事件链可回放、可审计

阶段二：Beta 联调（3–6 个月）

关键词：多设备、可解释、可部署

真实设备接入

  • ≥3 类设备：PV 逆变器 / BESS PCS / DG 控制器
  • ≥1 类负荷控制接口
  • 通信异常处理：断链、超时、降级

调度策略 V1（规则优先）

  • TOU / 峰谷规则调度
  • DG 启停规则（最小运行时间、最小负载率）
  • BESS 备用 SOC 保留
  • 24h 滚动计划 + 偏差记录

告警、事件与报表

  • 告警等级体系
  • 跳闸 / 合闸 / 启停 / 保护事件日志
  • 日报导出（能量 / 油耗 / 切换 / 故障）

权限与安全增强

  • 管理员 / 操作员 / 只读角色
  • 操作留痕
  • SSH / 白名单 / 防火墙 / 配置审计

区域插件首次落地

  • 网规 / 曲线 / 保护窗配置化
  • OpenADR / IEC 104 接口预留
  • 示例插件（二选一）：
    • IEEE 1547 参数模式
    • 中国峰谷 / 尖峰策略模板

试点准备

  • 点表 / 命令 / 互锁 / 责任边界清单
  • Shadow → Partial → Full 控制切换流程

阶段验收底线

  • HIL 全链路稳定运行
  • 调度执行结果与策略一致、可解释
  • 多故障场景下正确降级与告警
  • 权限与安全机制有效

阶段三：现场试点与 V1 冻结（6–12 个月）

关键词：敢签字、可复用、能卖

现场三阶段接管

  • Shadow Mode
  • Partial Control
  • Full Automatic（连续 ≥4 周）

可靠性与异常补齐

  • 黑启动验证
  • 再并网失败回退
  • 通信抖动自愈
  • 防误控 / 防越权 / 防非法合闸

策略与参数收敛

  • SOC / DG / 负荷参数标定
  • 可选轻量预测优化（不引入复杂 AI）

文档与合规启动

  • 安装 / 调试 / 运维手册
  • FAT / SAT 模板
  • UL / IEEE 或 CE 认证路径启动

产品化冻结

  • Base：Global Core
  • Add-ons：区域模块
  • 明确 V2+ 能力清单

试点验收底线

  • 切换 / 重联成功率与失败分类清晰
  • 节费与油耗降低可审计
  • 可再生消纳率提升
  • EMS 无宕机、无错控、无越权
  • 客户验收签字 + 事件回放报告

一句话总结

V1 的目标不是“功能最多”，而是：能跑、能控、能解释、能进现场、能卖。
