统一技术澄清要求

以下是《统一技术澄清要求（第1–2页汇总版）》的正式精简稿，适合直接发给设计院、业主或EPC：

统一技术澄清要求

（第1&2，3项）

一、电池系统（箱式储能）

  1. 表中“2000 MAh”应为2000 MWh或2000 kWh/套，请确认。
  2. 电芯314 Ah未说明化学体系（LFP/NCM）、循环寿命及额定电压范围。
  3. 消防系统类型及与BMS、EMS的联动逻辑未说明（气体/水喷/声光等需明确）。
  4. 冷却系统仅写“水冷机1台”，需确认是否具备N+1冗余及控制策略。
  5. 应补充泄爆、防火分仓、绝缘监测、通信协议（Modbus-TCP/IEC 104）等条款。

二、跟网型变流升压一体机（PCS+升压+主变）

  1. 电压描述“5.0/0.69 kV–36.5 kV”不清，请确认主变一次、二次额定电压及分接范围。
  2. “5250 kVA”为单台还是整套容量？应与PCS数量及额定功率对应说明。
  3. 确认执行标准：IEEE 1547-2018、UL 1741 SB、IEC 62271系列。
  4. 是否具备LVRT/HVRT、Volt-Var、Volt-Watt、Zero Export等并网功能。
  5. 通信与保护方案（IEC 61850或Modbus TCP）需明确。

三、EMS服务器柜

  1. “2.00 面”计量单位不合理，应改为“套”或“台”。
  2. 服务器配置未列，建议最低配置：双路64核CPU、256 GB内存、2 TB SSD RAID。
  3. 是否为主备冗余架构需说明，并明确是否含AGC/AVC调度接口。
  4. 通信协议（IEC 104、Modbus、MQTT）及网络安全要求（防火墙、VPN、日志审计）应写清。

（第10项）

四、能源管理监控软件

  1. 软件功能范围不明确，请确认是否包含监控界面、策略控制、日志与远程控制功能。
  2. 确认部署方式：本地EMS服务器安装或云端平台接入。
  3. “含控制策略”具体指哪些模块（TOU、需量优化、Zero Export等）需明确。
  4. 明确与PCS、BMS的通信协议（Modbus-TCP/IEC 104/MQTT）。
  5. 软件授权方式（永久/年费/节点授权）需说明。

（第12项）

五、UPS设备

  1. UPS额定功率未列，建议≥3 kVA，输出220 VAC 单相。
  2. 结构应为双变换在线式（Double Conversion），带自动旁路及过载保护。
  3. 电池建议采用锂电池组，容量满足满载2小时运行。
  4. 应配置通信接口（RS485或SNMP），可与EMS实现状态监测。
  5. 明确是否为各柜独立供电或集中供电。
