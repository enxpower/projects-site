执行摘要（供董事长/董事会3分钟决策）

本计划的关键不在“技术更强”，而在“合规更早、标准更准、合同更硬”。

我们建议以亿级美元预算，按“标准占位 → Utilities背书 → Colo复制 → Hyperscale进入”的顺序推进，并用“合同能源管理（EMC）+ 电网需求响应（DR）+ 可审计透明度协议”建立美国侧信任与采购通行证。

  • 市场逻辑：美国电网与许可周期正在成为数据中心扩张的瓶颈，企业开始把数据中心当作可调负荷签DR/削峰协议。Google已公开披露通过需求响应协议让数据中心更灵活以支持电网。 (blog.google)
  • 制度抓手：DOE等机构推动数据中心能效评估与专业认证体系（DCEP），意味着能效正在被制度化与采购化。 (Data Center Efficiency)
  • 标准入口：OCP Open Rack V3已形成机柜供电/连接器/电源架与母排接口文档体系，是“用标准语言进入美国供应链”的最佳入口之一。 (Open Compute Project)
  • 合规底线：涉及美国关键基础设施/敏感领域的投资与交易存在CFIUS审查不确定性，必须采用“先合规后扩张”的交易结构与可剥离设计。 (U.S. Department of the Treasury)

第一章：全球算力主权移交——从“算力芯片”到“算力能效”

1.1 2026年AI竞争的终点为什么是能源

AI训练/推理的工程本质是把瓦特（W）转成比特（bit）：

  • GPU/加速卡密度继续提升，使单机柜功率进入100kW+常态；
  • “上电容易、批电难、散热难、稳定性难”取代“买卡难”，成为扩张的硬约束；
  • 于是竞争焦点从“谁的芯片更强”转向“谁能在同样电力预算下产出更多有效算力（Watt-to-Bit Conversion Efficiency）”。

战略推论： 未来三年，美国市场会把“能效、可控、可调、可审计”当作比“更便宜的硬件”更硬的采购门槛。能效不仅是技术指标，更会进入许可、合同与监管语言。

1.2 中国长城的“国家级红利”如何转化为出海优势

长城不靠“计算生态”，只靠电力与能源系统能力，即可形成差异化：

  1. 电力电子规模化制造能力：高密度电源、磁材、功率模块的工程化与一致性控制；
  2. 系统级集成：HVDC供电 + 液冷热管理 + 短时储能/UPS + 电网友好接口的整合交付；
  3. 全生命周期运维能力：数据中心客户买的是“稳定性与可用率”，不是“设备清单”。

战略表达： 长城提供“算力基础设施能源底座（Energy Base for Compute Infrastructure）”，不触碰数据处理与内容安全边界，从而把地缘敏感度降到最低。

第二章：地缘政治“绕行战略”——应对不友好环境的物理布局

注意：本章是“政治工程学”，目标不是最便宜，而是最可通过。

2.1 物理布局总原则：轻资产在美、重制造在中、认证与服务外圈

  • 美国本土：轻资产、重就业（最终组装/测试/维保/培训/备件）
  • 中国：重制造、重供应链（核心功率模块、磁材、关键工艺一致性）
  • 加拿大：北美服务与可信认证缓冲层（培训、备件、第三方测试协同）
  • 匈牙利/新加坡：供应链中转与合规缓冲（高精度部件/特定物料中转、仓储与合规文件管理）

为什么要“美国本土终装（Final Assembly & Test）”

美国采购方在基础设施类项目的真实偏好是：

  • 本土可见的就业与税收贡献
  • 本土可控的交付与维保
  • 可审计的供应链透明度

    这与纯贸易进口的逻辑完全不同。

    同时，涉及敏感领域交易存在CFIUS等审查不确定性，越早把组织形态设计成“可解释、可审计、可剥离”，越能降低交易与合同风险。 (U.S. Department of the Treasury)

2.2 匈牙利与新加坡的角色：不是“制造替代”，而是“合规与交付缓冲器”

匈牙利（EU）建议定位：

  • 高精度连接器/机柜级关键结构件的仓储与中转；
  • 面向欧盟供应链合规的文件与追溯体系；
  • 作为“应对突发贸易/关税/禁运波动”的备份节点。

新加坡建议定位：

  • 东南亚区域备件与物流中枢；
  • 面向亚太客户的能效测试与培训中心（同时为北美输出培训体系）；
  • 作为国际金融与保险结构的中性节点（质保/性能担保/能效合同的再保险安排）。

2.3 为什么明确放弃墨西哥（董事会口径）

在“高能效基础设施”这一类项目中，墨西哥路径的主要价值是成本与近岸制造。但本计划的第一阶段目标是政治可通过性与信任建立：

  • 如果需要近岸制造，美国终装 + 加拿大服务已能满足“本土可控”诉求；
  • 墨西哥路径反而会把风险转移到跨境物流、合规复杂度与政治不确定性上；
  • 因此，阶段一明确不采用墨西哥，避免把管理复杂度引入早期“拿通行证”的关键窗口。

第三章：技术降维打击（技术总工视角）——以“瓦特-比特”转换律重建标准

本章只谈能源与热管理，不出现PKS/计算生态叙事。

3.1 新技术标准框架：Watt-to-Bit Conversion Stack（W2B栈）

我们把产品从“电源”升级为“W2B栈”，分四层定义：

  1. Power Conversion Layer（电能转换层）
    • 目标：98.5%级别的液冷高密度电源路线（分段效率目标与热设计余量）
    • GaN/SiC器件平台：实现高频、高功率密度、低损耗
  2. Thermal Transport Layer（热传输层）
    • 冷板/喷淋/浸没适配
    • 漏液监测、绝缘管理、介质兼容性、可维护性（MTTR指标）
  3. Ride-through Layer（抗扰动/短时能量缓冲层）
    • AI训练状态保存（Checkpoint)所需“秒级”能量缓冲
    • UPS一体化：不是长时备电，而是“业务连续性保险”
  4. Grid-Friendliness Layer（电网友好层）
    • DR/削峰接口、功率限制、分级降载
    • 全链路审计日志：把“可调负荷”变成可合同化资产
    • 这一层直接对应美国市场真实趋势：数据中心与公用事业签需求响应协议。 (blog.google)

3.2 “液冷电源黑盒”方案：合规黑盒、审计白盒

你之前的“黑盒”概念必须升级为：

  • 对外界：核心控制策略与器件选型为商业秘密（黑盒）
  • 对监管与客户：网络隔离、接口最小化、日志与审计项完全透明（白盒）

必须固化的三条工程规则：

  1. No IT Lateral Movement：能源系统不接入客户IT网，不成为横向移动通道
  2. Minimum Interface：与BMS/EMS/Utilities只通过最小必要接口对接，可随时物理/逻辑关断
  3. Audit Pack：出厂即提供第三方审计包（隔离证明、日志机制、固件签名、SBOM清单）

3.3 OCP语言兼容：用标准入口进入采购体系

OCP Open Rack V3对机柜、供电架、连接器、48V母排等已有系统规范与文档，是我们“标准占位”的重要入口。 (Open Compute Project)

策略：

  • 向上兼容 OCP 48V母排/电源架接口
  • 在兼容基础上做差异化：液冷供电架、短时缓冲BBU/超级电容模块、电网友好接口与审计包

3.4 “AI级防断电状态保存”能源系统：从硬件卖点变成财务卖点

AI训练的停机损失不是“设备损坏”，而是“训练中断导致的时间、算力与电费沉没成本”。

因此，短时能量缓冲/UPS一体化必须用财务语言表达：

  • “每次瞬断导致的平均损失 = 人工 + 训练时间 + 算力机会成本 + 电费沉没”
  • 我们的产品把瞬断从“业务灾难”降级为“可控事件”。

    这会成为COO在谈判中的核心筹码（见第四章EMC模式）。

第四章：商业模式与市场攻坚（COO视角）

4.1 EMC（合同能源管理）分账模式：用“省钱”换“准入”

基本结构：

  • 客户不为硬件一次性买单，改为签“能效提升与削峰能力”的长期合同；
  • 长城投入设备与改造，客户按“节省电费/容量费用/停机损失减少”进行分成；
  • 这把采购障碍从CAPEX预算争夺，转成OPEX可控与绩效对赌。

EMC合同的三类可计量KPI：

  1. 能效：PUE下降 / 电源转换效率提升 / 热管理能耗下降
  2. 电网：削峰响应能力（kW）、响应时间（秒/分钟）、持续时间（分钟）
  3. 可靠性：可用率、MTBF、MTTR、瞬断事件的业务影响减少

现实支撑：需求响应正在成为数据中心与Utilities合作的正式工具，Google已公开披露通过需求响应协议让数据中心更灵活以支持电网。 (blog.google)

4.2 进入Google/Nvidia/Meta供应链的“三阶梯计划”（不走捷径）

阶梯1：Utilities背书（6–12个月）

目标：先获得公用事业认可与试点，拿到“电网友好型”通行证。

  • 输出：DR/削峰试点协议 + 审计报告 + KPI实测数据

阶梯2：Colo复制（12–18个月）

目标：在Colocation运营商场景跑出“可复制的PUE达标包”。

  • 输出：不停机改造方法论、标准化交付包、维保体系
  • 这是规模化现金流与口碑的来源。

阶梯3：Hyperscale参考设计（18–30个月）

目标：进入云巨头的参考设计与限定场景（先从“能源与热管理模块”进入）。

  • 输出：Reference Design认可、定制化供电架/液冷接口适配、长期框架合同

  1. 组织层隔离：
